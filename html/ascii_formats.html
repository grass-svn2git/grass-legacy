GRASS5.0: brief description of ASCII formats

This description of ASCII formats supported by GRASS5 is for quick reference only. For full up-to-date description see GRASS 5 Programmer's manual

  • Sites 
    #any comment, can be inserted anywhere
name|mysoils.site
desc|3D soils data for location A
time|15 Jan 1999
labels|east north elevation ID pH Corg color
form||||#%%@
333444.55|2543367.01|22.45|#1 %6.6 %20.123 @brown
333445.05|2543377.21|32.46|#2 %5.6 %10.043 @gray
    for lat/long data 
    labels|longitude latitude ID elevation
75:33:55.34W|33:44:59.21N|#1 %225.2

    Notes:

    • sites data are stored in this ASCII format in GRASS database
    • comment and header lines are optional
    • elevation can be given as a 3rd dimension or an attribute
    • read more about how to define time in GRASS: Programmer's Manual Chap. 23 Datetime library
    • read more detailed description of sites format: GRASS 4.2(= 5.0) Site Record Format Specification

  • 2D Raster 
    north: 3980319.16466812
south: 3978824.85093895
east: 443960
west: 442296
rows: 747
cols: 832
10.30957 10.318124 10.201462 9.9399 9.566847.............

    Notes:

    • this format is produced by r.out.ascii and can be read by r.in.ascii, raster data are stored as binary in GRASS database
    • header is mandatory
    • first row starts at NW (upper left corner) and the raster is written west to east, north to south.
