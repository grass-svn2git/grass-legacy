How to release GRASS binaries and source code

$Date$

Note: This text contains *some* rules only applicable to the
      development coordinators (currently Markus Neteler, Radim Blazek).


1. Local work (basically at ITC-irst)

 o - Update VERSION file to release version
   - Create release branch
   
 o Create Changelog file
     CVSBRANCH=`cat CVS/Entries | grep AUTHORS | cut -d'/' -f6 | cut -b2-`
     echo $CVSBRANCH
     VERSION=6.x.y
     sh tools/cvs2cl.pl --follow "$CVSBRANCH"
     mv ChangeLog ChangeLog_$VERSION
     # now manually merge the pre-6.x.y changes into the ChangeLog_$VERSION
     # file (take from HEAD)
     gzip ChangeLog_$VERSION

 o reset VERSION file to CVS version

 o Store the source tarball in:
     http://grass.itc.it/grass/grass6x/source/
     along with associated files:
      NEWS.html ChangeLog_$VERSION SUBMITTING REQUIREMENTS.html INSTALL SUBMITTING

 o update web site to new version (HTML in Web-CVS)
      - main.inc
      - download/software.inc
      - grass6x/index.php
      - devel/grasshist.html
      - devel/grass_releases.html
      - bugtracking/bugreport.html
      
     write announcement
      - store in Web as announces/announce_grass6xx.html
